# 0.4.0 Release Notes

`0.4.0` is the first published raw-record ingress release for `causal-order`.

The headline change is `translateBatch()`: a narrow synchronous boundary for mapping arbitrary user-space records into the event envelope before ordering.

## Package Surface

Added:

* top-level `translateBatch()` as the first public synchronous raw-record ingress API
* public translation types covering:
  * translated immutable envelopes
  * typed mapper configuration
  * structured translation anomalies
  * translated-versus-rejected batch results
* direct unit coverage for the `0.4.0` ingress contract surface

Finalized:

* required mapper contract:
  * `getEventId`
  * `getNodeId`
  * `getPhysicalTime`
* explicit optional-mapper behavior, including deterministic `undefined` handling and field-local mapper exception surfacing
* relationship-field rules:
  * `parentEventId` is scalar-or-omitted
  * `dependencyEventIds` is array-or-omitted
  * scalar-to-array convenience coercion is not part of the published surface
* deterministic temporal coercion for `getPhysicalTime` and `getIngestedAt`
* structured `TranslationAnomaly` output with stable machine-readable metadata
* shallow translated-envelope immutability with `payload` preserved by reference

## Temporal Contract

Accepted timestamp inputs:

* `bigint`
* safe integer `number`
* canonical integer `string`

Rejected timestamp inputs include:

* `Date`
* ISO timestamp strings
* fractional numbers
* non-canonical numeric strings
* unsafe integers

The release also aligns core clock validation with that ingress contract by allowing negative epoch-millisecond `bigint` timestamps.

## Architectural Boundary

`0.4.0` intentionally keeps the ingress surface narrow.

It does not widen the core package into:

* CLI or terminal tooling
* filesystem-backed ingestion helpers
* JSONL or file-format adapters
* transport-specific connectors
* async translation APIs
* stream-translation or backpressure-aware ingestion helpers

The goal of the release is a defensible protocol boundary, not a broad environment-aware integration toolkit.
