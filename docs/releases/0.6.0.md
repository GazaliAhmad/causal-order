# 0.6.0 Release Notes

`0.6.0` is the first operational-tooling and integrations follow-through release for `causal-order`.

It builds on the published `0.5.0` contract boundary rather than reopening payload-aware or domain-resolution semantics.
It is now the published operational release line on top of that `0.5.0` boundary.

## What Landed

`0.6.0` now adds:

* the first public `inspect` helper layer:
  * `summarizeEventAnomalies()`
  * `summarizeTranslationAnomalies()`
  * `explainOrderedEvent()`
  * `inspectOrderResult()`
  * `inspectOrderBatch()`
* a focused `causal-order/inspect` subpath for narrower operational imports
* workflow guides for:
  * bounded replay inspection before downstream writeback
  * correction-capable streaming reconciliation follow-through
  * first-pass operator metrics and monitoring
* the first integration-shaped example for local durable-buffer replay using disk-backed JSONL staging

## Inspection Helper Direction

`0.6.0` does not turn the core package into a dashboard, projection engine, or workflow shell.

The helper layer stays intentionally additive:

* it summarizes or explains existing package output
* it does not mutate ordered results
* it does not hide anomalies
* it does not invent stronger causal certainty than the runtime already exposes

That means the new inspection layer is for:

* operator summaries
* replay audits
* emitted-batch inspection
* quick event-level explanation

It is not a hidden policy engine.

## Workflow Direction

The workflow guides now make the first `0.6.0` operational stance explicit:

* raw events stay source evidence
* anomalies stay queryable and operator-visible
* ordered rows and emitted stream batches are derived state
* correction-capable output must be treated as replaceable or superseding derived state rather than silent final truth

That includes separate playbooks for:

* bounded replay review and writeback posture
* correction-capable stream batch handling
* first metrics to count, graph, and investigate

## Integration Example Direction

`0.6.0` also adds a first integration-shaped example without turning the package into an adapter catalog.

The local durable-buffer replay example shows:

* disk-backed JSONL staging
* the public `translateBatch()` to `orderEvents()` path
* replay inspection via `inspectOrderResult()`

That keeps the example package-facing while still showing a realistic operational slice.

## Release Boundary

`0.6.0` still stays inside the payload-agnostic boundary clarified by `0.5.0`.

It does not add:

* official database connectors
* broker clients
* companion CLI or daemon tooling
* payload-aware contradiction resolution
* payload-aware fork resolution
* semantic dedupe policy execution inside the core runtime

The release is about operational usability and visibility, not about widening domain semantics.
