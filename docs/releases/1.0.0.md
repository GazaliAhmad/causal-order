# 1.0.0 Release Notes

`1.0.0` is the first stable public release of `causal-order`.

It does not widen the payload-agnostic core.
It publishes the already-hardened runtime as a stable contract, removes the last transitional root-only helper alias, and aligns the package-facing docs around stable adoption rather than pre-`1.0.0` preparation.

## What Landed

`1.0.0` adds:

* the first stable public contract for the bounded batch, streaming, translation, validation, and inspection surfaces
* the final helper-posture cleanup at the root import:
  * `compareClocks()` removed after the `0.9.x` deprecation line
  * `compareByHlc()` remains the direct HLC comparison helper
  * `compareDeterministically()` and `applyTieBreaker()` remain the supported deterministic fallback helpers
* stable release-surface alignment:
  * package metadata bumped to `1.0.0`
  * README, compatibility guidance, upgrade guidance, roadmap framing, and release history updated to speak from the stable line
  * generated website API metadata refreshed so the public reference surface matches the shipped package

## Stable Meaning

`1.0.0` means:

* the documented primary entrypoints are the intended long-term path
* breaking changes should be exceptional and explicit
* defaults, confidence labels, anomaly posture, and package-facing workflow guidance should not drift casually
* the package is ready for real production evaluation as a deployable event-ordering runtime

The stable release is therefore about preservation, clarity, and trust, not about inflating the scope of what the core package claims to solve.

## Release Boundary

`1.0.0` does not add:

* new contradiction-resolution semantics inside the payload-agnostic core
* built-in entity-fork or semantic-dedupe policy runtimes
* deployment-specific adapters or transport glue inside the main package
* new streaming truth claims beyond the existing watermark, lateness, correction, and reconciliation contract

The release keeps the core narrow on purpose while making that narrow contract safe to adopt as a stable dependency.
