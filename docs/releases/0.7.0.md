# 0.7.0 Release Notes

`0.7.0` is the transferability-baseline release for `causal-order`.

It keeps the runtime and API surface narrow while making the package easier to evaluate, maintain, and adopt without relying on undocumented project memory.

## What Landed

`0.7.0` adds:

* maintainer-facing release and compatibility docs:
  * `MAINTENANCE.md`
  * `RELEASE_PROCESS.md`
  * `COMPATIBILITY.md`
* public package-boundary guidance:
  * supported versus unsupported usage
  * upgrade expectations for the current preserved surface
* stronger discovery guidance:
  * examples and entrypoints guide
  * lightweight package-surface overview
  * clearer examples index positioning
* release-surface and docs-site polish:
  * automated publish-size badge generation
  * tighter public-site filtering for repo-only docs
  * RSS, sitemap, canonical, and share-image support
  * more consistent guides, wiki, and API layout behavior

## Transferability Direction

`0.7.0` is about making the package easier to carry forward safely.

That means:

* a new maintainer can find the release process without asking for oral history
* a reviewer can tell which compatibility boundaries matter
* a user can tell what is supported, unsupported, or intentionally out of scope
* an evaluator can find the right example or public entrypoint without reading the whole repo first

## Release Boundary

`0.7.0` does not widen the core runtime claim surface.

It does not add:

* new ordering semantics
* new translation semantics
* new streaming guarantees
* new environment-specific adapters
* domain-aware contradiction or dedupe policy inside the payload-agnostic core

The release is about clarity, transferability, and package-facing guidance rather than runtime expansion.

## Release Result

The published `0.7.0` line now reads consistently across:

* `README.md`
* `CHANGELOG.md`
* `ROADMAP.md`
* `CONTRIBUTING.md`
* `MAINTENANCE.md`
* `RELEASE_PROCESS.md`
* `COMPATIBILITY.md`
* `guides/README.md`
* the website guides, wiki, and API surfaces

At this stopping point, `0.7.0` is the first release where the package surface, public guidance, and maintainer guidance all read like one coherent product line.
