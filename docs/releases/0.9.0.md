# 0.9.0 Release Notes

`0.9.0` is the published final stabilization release for `causal-order`.

It does not widen the core runtime.
Instead, it turns the already-maturing public surface into a more preservable contract by cleaning up helper posture, freezing the core-versus-policy boundary, and aligning the package-facing docs and website around what the library actually is today.

## What Landed

`0.9.0` adds:

* final pre-`1.0.0` contract cleanup across the runtime-facing API:
  * `compareWithTieBreaker()` removed from the public surface
  * `compareClocks()` reduced to deprecated root-only compatibility surface through `0.9.x`
  * `applyTieBreaker()` preserved as an intentional low-level public helper
  * `orderValidatedEvents()` narrowed to the public validated-events-plus-options shape
  * `StreamOrderBatch` adopted as the final stream-result type name
* final pre-`1.0.0` boundary freeze across the higher-layer policy seam:
  * extension-policy interfaces preserved as intentional public boundary types
  * public wording tightened so those interfaces are no longer read as accidental draft spillover
  * `strict`, `detectAnomalies`, and `allowUnknownOrder` frozen around one consistent explanation
* release-surface and website alignment:
  * README, guides, and generated API wording aligned with the landed runtime and boundary decisions
  * website framing tightened around `causal-order` as a deployable event-ordering runtime
  * structured-data and crawler-facing metadata updated so the software surface reads more like a deliberate product line

## Stabilization Direction

`0.9.0` is the release where the project stops leaving important public-surface questions in a "probably later" state.

That means:

* primary names and compatibility posture are now explicit
* the payload-agnostic core-versus-policy split is now explicit
* the package-facing docs now teach one consistent operational story instead of a mix of historical and provisional wording

The release is therefore about preservation and low-ambiguity contract cleanup, not about widening semantics.

## Release Boundary

`0.9.0` does not add:

* new causal-ordering semantics
* new translation semantics
* new streaming guarantees
* built-in contradiction, fork, or semantic-dedupe runtime inside the payload-agnostic core
* a final plug-in runtime or extension lifecycle framework
* deployment-specific adapters or environment glue inside the main package

The release is about making the current package easier to trust, preserve, and evaluate as a real published runtime contract.
